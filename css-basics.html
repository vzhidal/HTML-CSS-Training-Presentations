What?

  • CSS is an acronym for Cascading Style Sheets.
  • CSS is a style language that defines visual styles of HTML documents.
формальный язык описания внешнего вида документа, написанного с использованием языка разметки.

Why?

  • Separation of presentation from markup.
  • Cleaner code.
  • Easier to manage style changes.
отделение представления от разметки. более чистый код, который проще сопровождать и расширять.

В начале 1990-х различные браузеры имели свои стили для отображения веб страниц. HTML развивался очень быстро и был способен удовлетворить все существовавшие на тот момент потребности по оформлению информации, поэтому CSS не получил тогда широкого признания.

Термин «каскадные таблицы стилей» был предложен Хокон Виум Ли в 1994 году. В это время уже существовало большое разнообразие языков стилизации. Однако из-за концепций, на которых основан css, именно он набрал наибольшую популярность.

17 декабря 1996 года css level 1 получил статус рекомендации. Что же представлял из себя css 1? Параметры шрифтов. Цвета. Атрибуты текста. Свойства блоков.

css level 2 Рекомендация принята 12 мая 1998 года. Блочная вёрстка. Расширенный механизм селекторов. Генерируемое содержимое.

Уровень 2, ревизия 1 принята 7 июня 2011 года. CSS2.1 основана на CSS2. Исправлены ошибки, некоторые модули удалены.

CSS3

  • CSS 3 is divided into several separate documents called "modules".
  • Each module adds functionality and/or replaces part of the CSS2.1 specification.

Термин CSS3 относится ко всему, что было опубликовано после css 2.1. CSS3 не является стандартом и представляет из себя группу модулей объеденённых под одним названием. Разрабатываются и релизятся отдельно. Каждый модуль может получать новый уровень независимо от других. Самостоятельной спецификации «CSS3» нет и никогда не будет. Долгие годы название «CSS3» было символом прогресса и светлого будущего веб-технологий. Что же нас ждёт дальше?

Value Processing

Value Processing

A user agent must assign a value to every property that applies to the target media type to every box in the formatting structure.

Calculation

  • Collecting all the declared values applied to an element.
  • Cascading yields the cascaded value.
  • Defaulting yields the specified value.
  • Resolving value dependencies yields the computed value.
  • Formatting the document yields the used value.
  • Transforming to the actual value based on constraints of the display environment.

Каждое свойство примененное к элементу становится declared value
При помощи каскадирования браузер получает "wining value"

cascaded value представляет из себя результат каскадирования. (declared value которое выйграло каскадирование)
Если результатом каскадирования является пустой список, то не существует cascaded value.

specified value является рзультатом пропускания cascaded value через процесс выставления значения по умолчанию. Этот процесс позволяет убедиться, что каждое свойство, каждого элемента будет иметь своё значение.
Во многих случаях specified value является cascaded value. Только в случаях, когда cascaded value не существует оно будет заменено значением по умолчанию.

computed value является результатом обработки specified value. Для каждого свойства и значения существует алгоритм вычисления computed value.Восновном все алгоритмы сводятся к преобразованию значения свойства в абсолютное значение и подготовки его к наследованию. Является значением, наследуемым от предка к потомку. values with relative units must be made absolute by multiplying with the appropriate reference size

  • значения в относительных единицах (em, ex, vh, vw)должны быть преобразованы в абсолютные
  • определённые ключевые слова(smaller, bolder) должны быть заменены значениями
  • процентные значения некоторых свойств
  • относительные url должны быть преобразованы в абсолютные

computed value вычисляется из specified value по мере возможности без формирования документа или обработки других сложно-вычисляемых операций (таких как получения значений самого элемента или его родителя, или обработка сетевых запросов)
computed value существует даже если css свойство не применимо к текущему элементу, т.к. другие свойства могут вычисляться на их основе.

used value является результатом окончательных вычислений, которые были отложены на этапе вычисления computed value. Является абсолютным значением свойства, теоретически используемым в отображении документа. Если свойство не применимо к элементу, то элемент не будет иметь used value для данного свойства.
Например, width: auto не может быть вычисленно без информации о расположении предков элемента. Поэтому computed value - auto, а used value абсолютное значение, такое как 100px
(if a property does not apply to an element, it has no used value; so, for example, the flex property has no used value on elements that aren’t flex items.)

used value теоретически может быть использовано, но не всегда user agent может отобразить его на текущем оборудовании (only be able to render borders with integer pixel widths). Поэтому должны быть вычислены приблизительные значения. actual value является тем значением, которое будет использовано после вычисления всех приближений.

Filtering

Filtering

In order to find the declared values, user agents must first identify all declarations that apply to each element.

Чтобы найти все declared values юзер агент должен идентифицировать все объявления связанные с этим элементом

Filtering

  • A declaration applies to an element if:

    • It belongs to a style sheet that currently applies to this document.
    • It is not qualified by a conditional rule with a false condition.
    • It belongs to a style rule whose selector matches the element.
    • It is syntactically valid: the declaration’s property is a known property name, and the declaration’s value matches the syntax for that property.
  • Он пренадлежит к таблице стилей применённой к документу
  • Он объявлен в условном правиле, имеющем положительное состояние
  • Он принадлежит правилу, имеющему селектор, совпадающий с элементом.
  • Он синтаксически правильный.

Cascading

The cascade takes a unordered list of declared values for a given property on a given element, sorts them by their declaration’s precedence, and outputs a single cascaded value.

Cascading order

Which CSS rules “win”?

There are four steps to determine which CSS rules will “win”.

Существует 4 шага для определения какое css свойство будет применено к элементу.

question

Step 1

Gather all the declarations that apply to an element and property from browser, author and user style sheets.

прежде всего, браузер собирает все объявления применённые к css свойству элемента, включая: стили браузра, пользовательские стили и авторские стили.

Step 2

  • Sort the gathered declarations according to:

    • origin
      • user agent
      • author
      • user
    • importance
      • normal
      • !important
На втором шаге все собранные объявления сортируются по происхождению и уровню важности.

From lowest to highest priority

  • Normal declarations in user agent style sheet.
  • Normal declarations in user style sheet.
  • Normal declarations in author style sheet.
  • !important declarations in author style sheet.
  • !important declaration in user style sheet.
  • !important declaration in user agent style sheet.

Step 3

If declarations have the same origin or importance then the declaration’s selectors need to be scored, to see which declaration will “win”.

Если объявления имеют одинаковое происхождение и уровень важности, то браузер вычисляет специфичночть селекторов.

Selectors Specificity

  1. Inline style.
  2. Count the number of IDs.
  3. Count the number of classes, attributes and pseudo-classes.
  4. Count the number of element names or pseudo-elements.

A note on concatenation

“A” will always beat “B”, which will always beat “C”, which will always beat “D”.

Step 4

If two declarations have the same importance, origin and specify, the later specified declaration wins.

Если объявления имеют одинаковое происхождение, уровень важности и специфичность, то последнее объявление будет приоритетным.

Defaulting

  • When the cascade does not result in a value, the specified value must be found some other way: 

    • Inherited properties draw their defaults from their parent element through inheritance.

Collapsing restrictions

  • Only margins of block-level boxes can collapse.
  • Margins of a floated box do not collapse with any other margins.
  • Margins of a box with ‘overflow’ other than ‘visible’ do not collapse with its children's margins.
  • Margins of an absolutely positioned box do not collapse with any other margins.
  • Margins of the root element's box do not collapse.

    • inline-blocks;
    • table cells and table captions;
    • elements where overflow has a value other than visible;
    • flex boxes.
Внутри него происходит формирование блочных боксов и взаимодействие floats

Absolute positioning

  • Boxes are entirely removed from the flow.
  • Boxes don't interact with the flow at all.
  • Boxes positioned relative to their containing block.
боксы полностью удаляются из потока формирующего позиции элементов и не взаимодействуют с ним. Они позиционируются относительно их containing block.

Containing Block

Element boxes are positioned within a formatting context, which, by default, is provided by the box generated by a parent element.

элементы позиционируются в контексте форматирования, который по умолчанию генерируется родительским элементом.

containing block

position: static or relative

  • The containing block is formed by the edge of the content box of the nearest ancestor element whose display property value is one of:
    • block
    • inline-block
    • list-item
    • run-in
    • table
    • table-cell
для static и relative элементов containing block формируется внутри content edge ближайшего предка с со следующим типом:

containing block

position: absolute

  • The containing block is the nearest positioned ancestor (the nearest ancestor whose position property has one of the values absolute, fixed, or relative).
  • The containing block is formed by the padding edge of that ancestor.
для абсолютно позиционированных элементов containing block формируется внутри padding edge ближайшего позиционированного предка

containing block

position: fixed

  • The containing block is the viewport (for continuous media) or the page box (for paged media).
для фиксированно позиционированных элементов containing block формируется внутри viewport

The stacking context

  • Boxes are positioned in three dimensions.
  • The third dimension is the z axis, which is perpendicular to the screen.
боксы позиционируются не в 2-ух а в трёх измерениях. Третье измерение задаётся при помощи свойства z-index и располагается перпендикулярно экрану.

'z-index' property

  • For a positioned box, the 'z-index' property specifies:

    • The stack level of the box in the current stacking context.
    • Whether the box establishes a local stacking context.
для позиционированных боксов z-index задаёт уровень стека отображения в текущем контексте. А также задаёт локальный стек.

Forming of a stacking context

  • The root element (HTML).
  • Positioned (absolutely or relatively) with a z-index value other than "auto".
  • A flex item with a z-index value other than "auto".
  • Elements with an opacity value less than 1.
  • Elements with a transform value other than "none".
  • Elements with a mix-blend-mode value other than "normal".
  • Elements with a filter value other than "none".
  • Elements with isolation set to "isolate".
Browser specific and newest issues: on mobile WebKit and Chrome 22+, position: fixed always creates a new stacking context, even when z-index is "auto" (See this post) specifing any attribute above in will-change even you don't write themselves directly (See this post) elements with -webkit-overflow-scrolling set to "touch“

Stacking context layers

  • The background and borders of the element that establishes the stacking context.
  • The stacking contexts of descendants with negative stack levels.
  • Block-level descendants in the normal flow.
  • Floated descendants and their contents.
  • Inline-level descendants in the normal flow.
  • Positioned descendants whose z-index is auto or 0.
  • The stacking contexts of descendants with positive stack levels.

stacking context

(summary)

  • Positioning and a z-index value creates a stacking context.

  • Stacking contexts are hierarchical.

  • Each stacking context is completely independent from its siblings.

CSS At-rules

CSS At-rules

  • @charset
  • @import
  • @media
  • @page
  • @font-face
  • @charset применяется для задания кодировки внешнего CSS-файла.
  • @import позволяет импортировать содержимое CSS-файла в текущую стилевую таблицу.
  • @media позволяет указать тип носителя, для которого будет применяться указанный стиль.
  • @page позволяет задать значение полей (отступов) при печати документа или для страниц, у которых тип носителя задан как print.
  • @font-face позволяет определить настройки шрифтов, а также загрузить специфичный шрифт на компьютер пользователя.

https://developer.mozilla.org/en-US/docs/Web/CSS/Shorthand_properties

Shorthand Properties

(Edge Cases)

A value which is not specified is set to its initial value. (it overrides previously set values).


div {
    background-color: red;
    background: url(images/bg.gif) no-repeat top right;
}
Значение, которое не указано устанавливается в исходное значение. (это перекрывает ранее установить значения)

Shorthand Properties

(Edge Cases)

  • Only the individual properties values can inherit.

  • The keyword inherit can be applied to a property, but only as a whole, not as a keyword for one value or another.


.parent {
  background: red;
}
.child {
  background: inherit;
}
только отдельные css свойства могут наследоваться. ключевое слово inherit может быть применено только к свойству целиком.

Shorthand Properties

(Edge Cases)

Shorthand properties try not to force a specific order for the values of the properties they replace.

  font: italic bold 12pt/10pt serif

сокращённые записи свойств по возможности не определяют порядок для значений свойств.
