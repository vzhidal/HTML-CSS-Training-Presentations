Comments for defining structure


.container {
    margin: 10px;
    padding: 10px;
}
/****** HEADER *******/
.header {
    background: red;
}
/****** END HEADER *******/

/****** BODY *******/
.content {
    background: red;
}
/****** END HEADER *******/

/****** FOOTER *******/
.footer {
    background: red;
}
/****** END FOOTER *******/

Results

  • A bit easier to maintain, just because it’s easier to find some logical parts
  • There is a big chance to place a style to a wrong group.

Impulse

  • Place logical parts of your styles in a separate files
  • Performance optimizations shouldn’t affect a project structure
  • It’s possible to use automatic build tools instead of developing in one file

Deep nesting for avoiding side effects

(One line styles)

.container div {margin: 10px;}
.container div .container ul {margin: 25px; padding: 10px;}
.container div .container ul li a span {margin: 20px; padding: 10px;}

Results

  • Big problems with VCS
  • Hard to maintain

Impulse

  • I can make my project more structural and maintainable with some modules.
  • Naming and semantic is really important.

Think in modules

  • Make it possible to reuse your modules.
  • Write container independent CSS.
